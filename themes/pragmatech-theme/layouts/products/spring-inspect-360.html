What makes this different

Three reasons this isn't another SonarQube screenshot.

Real pipeline data, not YAML peeking.

Most audits read your CI config and call it done. This one pulls live GitHub Actions or Jenkins API data — median build time, p95 duration, success rate, top slow jobs and steps, recent failures. The pipeline section reflects what your team actually experiences.

Hotspots are about where bugs live, not where lines change.

The audit uses code-maat (Adam Tornhill's industry-standard tool) to join churn with complexity. The result is a top-10 of files that are both hard to read and frequently touched — the actual bug factory. Not a list of "biggest files".

Non-invasive. Verified on every run.

The tool captures git status before and after each scan. If a single byte changed in your repo, the run exits with an error. Your code is yours.
