Frequently Asked Questions

Common questions about the Velocity Blueprint audit

Which languages and frameworks do you support?
We focus exclusively on Java and Spring Boot teams. This specialization is what allows us to deliver deep, actionable insights rather than surface-level generic advice. If your stack is Spring Boot (with Maven or Gradle), we're the right fit.
Does it matter which CI pipeline we use?
No. Whether you run Jenkins, GitHub Actions, GitLab CI, Bitbucket Pipelines, or any other CI system — the audit focuses on your build and test behavior, not the pipeline tooling itself. We analyze build times, test execution patterns, and context loading regardless of where they run.
Do you need access to our source code?
For a full-blown audit, yes — we need read access to your repository. This allows us to analyze test structure, context configurations, flaky patterns, and AI-generated code at the source level. We sign an NDA before any access to protect your intellectual property. Your code never leaves your infrastructure; we work within your environment.
What team size does this work for?
The Velocity Blueprint is designed for teams of 5–50 developers working on one or more Spring Boot services. Smaller teams get tactical quick wins; larger teams benefit most from the structural analysis and the 90-Day Roadmap.
What happens after we receive the report?
You get a 60-minute walkthrough call where we review every finding together. After that, your team can execute the 90-Day Roadmap independently. If you want hands-on help implementing the recommendations, we offer follow-up consulting engagements — but there's zero obligation.
We have a monorepo / multiple microservices — can you handle that?
Yes. The fixed-price audit covers one primary service or module. If you have a monorepo with multiple build modules or a microservices setup, we'll focus on the service with the biggest bottleneck first. Additional services can be audited as follow-up engagements.
